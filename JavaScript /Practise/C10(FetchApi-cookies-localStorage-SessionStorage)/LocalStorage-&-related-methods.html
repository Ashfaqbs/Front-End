localStorage & related methods

Local storage helps in storing the data of a particular domain,Same also stores the data as cookie as KEY and value pair

Local Storage is a web storage object which are not sent to the server with each request, Cookie is sent to the server with every request, but sometimes we dont want to send the data to the server then we use Local storage

This data survives a full page refresh and even a full browser restart , the data will remain even if we refresh or restart the browser
eg: we want to save the user data , but we dont want to save the data and send to server everytime so we will use localStorage and not cookie 

    
        


//Setting a localStorage or we can directly run in browser console
            localStorage.setItem('name','sonu')

            //Getting a item saved in localStorage by the key 
            localStorage.getItem('name')
'sonu'

even we refresh we will get the data , and localStorage key and value pairs are seen in browser> application> Storage >localStorage and our details will be seen

copy the url where you set the localStorage in the browser i.e http://127.0.0.1:5500/Practise/C10(FetchApi-cookies-localStorage-SessionStorage)/LocalStorage-&-related-methods.html

and now close the browser and open and paste the url you can see the details in localStorage


Important Note

1.Both key and value must be Strings 
2.We can use the two JSON methods to store objects in localStorage
JSON.Stringify(object)->Converts objects to JSON Strings
JSON.Parse(String) -> Converts String to objects(Must be a valid JSON)
