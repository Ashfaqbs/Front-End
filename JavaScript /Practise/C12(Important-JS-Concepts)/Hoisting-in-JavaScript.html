Hoisting in JavaScript

Hoisting refers to the process where by the interpreter appear to move the declarations to the top of the code before execution Variables can thus be referenced before they are decided in JavaScript
Important Note: JavaScript only hoists declarations , not initializations The variable will be undefined until the line where its initialized is reached and Functions expressions and class expressions can not be hoisted

Functions

  • normal function
    • can be called before or after declaring the Function demo()//hello from above function demo() { console.log("hello from above") } demo() //same o/p
  • Anonymous/Arrow function(Functions expressions and class expressions can not be hoisted)
    • We can Anonymous / arrow Functions by let or const can only be called after initializing as we declare the Functions variable with let or const demo1()//: Cannot access 'demo1' before initialization let demo1 = () => { console.log("hello from above by arrow function") } // demo1() // hello from above by arrow function
